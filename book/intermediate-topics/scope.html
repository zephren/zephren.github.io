Scope

Scope

If you feel like you have a grasp on functions, then the next topic to tackle is scope. Scope is a difficult concept to explain with words, so it is probably most easily demonstrated by a few examples:

// This variable is global and in the global scope
var aGlobal = 42

// Functions create a new scope when they runs
function myFunction() {
    // This variable only exists inside this function
    var aLocal = 'I am local to this function'

    // This variable has the same name, but it's value
    // only exists inside this function because we're
    // using "var"
    // It does not overwrite the value of "aGlobal" that is declared above
    var aGlobal = 'I am also local to this function'

    console.log(aLocal)
    console.log(aGlobal)
}

// This function also creates it's own scope when it runs
function modifyAGlobal() {
    // We can modify the global variable by not declaring
    // a new "var"
    aGlobal = 777
}

myFunction()

// Notice how aGlobal is still 42 here
console.log(aGlobal)

modifyAGlobal()

// Now notice how aGlobal has been modified
console.log(aGlobal)

The above code demonstrates how each function creates it's own scope, and has access to the scope "above it". Scope is relative to the place where the function is defined. This means it doesn't matter where you call a function from, as it will always have access to the scopes above it.

Here is another example to show this a little differently, with one function calling another.

// These variables are in the global scope
var varA = 1
var varB = 2

// These three functions are in the global scope
function functionA() {
    var varC = 'C'
}

function functionB() {
    var varD = 'D'
}

function functionC() {
    var varE = 'E'

    functionA()
}

Functions A, B, and C are defined within the global scope, so they are tied to that scope.

1) If we call functionA, then it has access to the variables varA, varB, and, in it's own scope varC. 2) If we call functionC, then it has access to the variables varA, varB, and varE. You can also see that it will call functionA... but functionA behaves the same as in #1. It does not have access to anything within functionC, just varA, varB, and varC.

Functions have no knowledge about the variables in other functions. Unless you define a function within another function (which we did earlier in the "functions" section):

// This function takes an array and adds up all the values,
// multiplying each value by the length of the array
// I don't know why you would want to do this, but it demonstrates the point
function weirdMath(theArray) {
    var total = 0;

    // This function only exists within the scope of weirdMath
    function addValueTimesLength(value) {
        total = total + value * theArray.length
    }

    for (var i in theArray) {
        addValueTimesLength(theArray[i])
    }

    return total
}

var myArray = [1, 2, 3, 5, 8, 13]
console.log(weirdMath(myArray))
