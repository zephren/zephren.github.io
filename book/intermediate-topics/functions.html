Functions

Functions

Up until now if you have been playing with JS on your own you have probably found that you were repeating yourself a lot. Maybe checking the same conditions in an if statement in multiple places throughout your code, or logging the same string with multiple variables in more than one place. This is a (non-medical) condition called is called code duplication, and the best cure for it is functions.

Functions allow you to break your code up into logical, reusable sections. You also get to give these sections names. When naming a function, it should be something that describes what it does and is hopefully easy to remember.

Function Syntax

Structurally, functions look a little like a single if statement, but have a little more to them and are much more powerful. Let's get some terms out of the way. When you write a function, that is called "defining" a function. And when you want to use a function and have it do something, that is called "calling" a function.

// Defining a function
function sayHello() {
    console.log('Hello!')
}

// Calling the function
sayHello()

If you run this example, you'll see that it simply outputs "Hello!". You might be thinking "well this seems excessive, all it is doing is logging out a string", and you would be right, it is excessive (unless of course you want to be super friendly and say hello a bunch of times, then it might seem a little more practical). However, the point here is to demonstrate the bare minimum to create a function an have it produce a tangible result. The key here is to realize that you can put anything you want inside a function and then call that function as many times as you would like. This enables you to write some code once and use it as many times as you want, by simply calling your function. Before moving on and making things more interesting, let me give you another metaphor.

Try to think of a task you do regularly in your every day life. Give it a name, and think of the inputs it requires. These would be materials, tools, or information required to perform your task. Bonus points if it has some sort of result or product... an output. Also think about the steps involved in your task. Keep this task in mind, along with it's inputs, steps, and outputs, as we go through the next few examples.

Now let's make the above function a little more interesting:

// Defining a function with a single parameter: whatToSay
function saySomething(whatToSay) {
    console.log(whatToSay)
}

// Calling the function with a parameter value of 'Hello!'
saySomething('Hello!')

// Calling the function with a parameter value of 'How are you!'
saySomething('How are you?')

// Calling the function with a parameter that is a variable
var jackSays = 'I can jump!';
saySomething(jackSays);

Now we have added what is called a function parameter; this would be one of the inputs to the task I had you think through. Parameters allow you to create variables that are special to the execution of your function, and you can have as many as you would like. If your function has parameters, you need to give those parameters values when you call your function. In the case of the above example, the function seems to expect a string for the first parameter. I say seems because we can pass whatever value(s) we want into any function

Here is a slightly more practical function (that seems to accept numbers):

function add(numberA, numberB) {
    return numberA + numberB
}

var result1 = add(2, 3)
var result2 = add(10, 10)

console.log(result1)
console.log(result2)

Now we have two parameters and a return value. The return value is the output to your function, which you are free to do whatever you wish with, like assign it to a variable. The return value is completely optional, and if you do not specify one then it defaults to a value of "undefined".

That is essentially all there is to functions. A bit (or sometimes a lot) of code that you can reuse wherever you please. It accepts any number of inputs that you choose and and has an optional output. Here are a few more examples:

// A function to average an array
function averageArray(array) {
    var total = 0;

    // Total all the items in the array
    for (var i in array) {
        total = total + array[i];
    }

    // Divide the total by the number of items
    return total / array.length;
}

// Creating an array
var myArray = [2, 5, 6, 9, 17]

// There's no need to assign it to a variable
// if all that is needed it to output the result
console.log(average(myArray))

Oh, and one last thing. Remember how everything is a value? Yep, you can assign functions to variables to:

// Defining what is called an anonymous function
// and assigning it to a variable 
var myFunction = function() {
    // It does a thing...
}

myFunction();

And pass functions into other functions:

var ifEven = function(number) {
    console.log(number + ' is even')
}

var ifOdd = function(number) {
    console.log(number + ' is odd')
}

var checkEvenOdd = function(number, callIfEven, callIfOdd) {
    if (number % 2 === 0) {
        callIfEven(number)
    } else {
        callIfOdd(number)
    }
}

checkEvenOdd(5, ifEven, ifOdd)
checkEvenOdd(42, ifEven, ifOdd)
checkEvenOdd(1987, ifEven, ifOdd)

This is definitely a contrived example, but it is meant to demonstrate the flexibility of JS.

Functions in Functions

In JS you can nest functions within functions (...within functions, ad infinitum). This will be important when learning about scope.

function logThings() {
    function simpleLog(string) {
        console.log(string);
    }

    simpleLog('This')
    simpleLog('Is')
    simpleLog('How')
    simpleLog('Functions')
    simpleLog('Within')
    simpleLog('Functions')
    simpleLog('Work!')
}

logThings()

// This will intentionally fail, to demonstrate that the inner 
// function is not available outside it's container function
simpleLog()

One last point to make about everything being a value. You can return functions from other functions, which will make more sense once you understand scope better. Here we are calling a function to create and return another "customized" function. You can then use this function anywhere you have access to the variable to which it is assigned.

function createAnAdder(number1, number2) {
    // This returns a function that will return the added
    // values of the numbers that are passed in
    // If this is too confusing, the scope section should
    // help clear it up
    return function() {
        return number1 + number2
    }
}

// Assign a function to "myAdder" that adds 3 and 4 any time you call it
var myAdder = createAnAdder(3, 4);

console.log(myAdder());
console.log(myAdder());
console.log(myAdder());
