Objects

For the first part of this section we will use the term "box" in place of a structure that is called a "JavaScript object".

Imagine you have a box. But this is a special box. You can put whatever you want inside this box, and It can be instantly resized to whatever size you need it to be.

This is a box (it's empty):

{}

You can assign your box to a variable:

var box = {};

Remember how this box can instantly be resized to any size you need? Let's see that in action. The best part is that you don't have to do anything other than put things in the box. And label them of course... you don't want to forget what's in the box. Feel free to literally imagine labels on the things you put in the box. Here is the basic structure of a box:

{
    'label': 1234,
}

Boxes are great for structuring and organizing information, and can contain pretty much whatever you want:

A string:

{
    'aString': 'Hi! I'm a string!',
}

Some numbers:

{
    'anInteger': 42,
    'aFloat': 3.14,
}

A boolean:

{
    aBoolean: true
}

Other boxes:

{
    'anEmptyBox': {},
    'aBoxWithStuff': {
        'aString': 'Hi! I'm a string!',
    'anInteger': 42,
    'aFloat': 3.14,
    },
}

An array:

{
    'anArray': [1, 2, 3, 4, 5],
}

Even functions:

{
    'add': function(a, b) {
        return a + b
    },
}

Here is a practical example of a what you might organize in a box (using a toy box as the example):

var toyBox = {
    'lego': {
        '2x2': 50,
        '2x4': 75,
        '1x2': 30,
        '1x4': 40,
    },
    'dolls': {
        'ken': {
            'hair': 'brown',
            'eyes': 'blue',
        },
        'barbie': {
            'hair': 'blonde',
            'eyes': 'brown',
        },
        'elmo': {
            'hair': 'red',
            'eyes': 'black',
        },
        'giJoe': {
            'hair': 'black',
            'eyes': 'black',
        }
    },
    'vehicles': [
        'Race Car',
        'Truck',
        'Plane',
        'Helicopter',
    ]
}

At this point we will start referring to "boxes" by their correct term, "objects". And "labels" and "things" will be referred to as "properties" and "values". Also, properties do not require quotes around them (unless you want to have a property that is a JavaScript keyword, or starts with a number, or contains an operator), so they won't be used unless necessary.

Objects are very flexible, but have a very basic structure as covered above. Something worth playing with is how you like to visually structure your objects. The format used above is pretty common, but there's nothing stopping you from putting it all on one line (word wrap prevents it from all being on a single line here, but try copying it into your code editor):

{'anEmptyBox': {}, 'aBoxWithStuff': {'aString': 'Hi! I\'m a string!','anInteger': 42, 'aFloat': 3.14}}

Now that you have some objects to work with the next question is probably, "How do I get stuff out of a box/object?". It's actually very simple, let's use an example from above. But before jumping into that it is worth mentioning that there are two ways to access the value of a property. Either using dot notation or array "index" notation. Array notation can be used to access any property, while dot notation cannot (specifically properties that start with numbers and/or contain operators).

var myObject = {
    anInteger: 42,
    aFloat: 3.14,
}

To get a value from it (using dot notation):

var x = myObject.anInteger;

"x" would now have the value of 42.

And for a more complex example using the toyBox from above (referencing inner objects):

var x = toyBox.dolls.ken.hair; // Dot notation
var x = toyBox['dolls']['ken']['hair']; // Array index notation

"x" is now "brown"

And for using both dot notation and array index notation:

var x = toyBox.lego['2x4'];

"x" is now 75.

Array index notation is required for '2x4' here because it starts with a number. The following is NOT VALID syntax:

var x = toyBox.lego.2x4; // This will fail to execute

It is also possible to access an object's property using a variable and array index notation:

var a ='2x4';
var x = toyBox.lego[a];

"x" is again 75

The following is valid syntax, but will not give you the value that toyBox.lego[a] will:

var a ='2x4';
var x = toyBox.lego.a; // "a" is not a known property of toyBox.lego

Now that you've seen how to access a value in an object, the next step is to add a property to an object or update a property on an object. This is done simply by placing the object access on the left hand side of the equals operator.

To update a property:

toyBox.lego['1x2'] = 50;

To add a property:

toyBox.lego['1x6'] = 10;
