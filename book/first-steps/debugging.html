Debugging

Debugging

This will inarguably be the most valuable tool of any programmer. No one writes perfect code that works the first time and that has no flaws. In any language, every programmer will introduce bugs into the code they write. This is just a fact of life. There is a ray of hope however, and that ray is the ability of a programmer to be able to debug their (and others) code.

Hopefully you've been playing with writing some of your own code up to this point, and even more hopefully you've been using console.log to see what it is doing. Even more points if you've run into an issue, and used console.log to help you figure out what was going wrong (by logging out something you might not normally have).

If you have not happened to try this out, then I hope you find the next few sections enlightening into the ways of debugging and benefit of mastering it as a skill.

What is a Bug

For as long as I have been writing code, I will still find myself introducing bugs relatively frequently. Most of the time they will be caught through manual and automated testing. As I write this, I am trying to come up with some examples of bugs, but finding it difficult to intentionally introduce one that might be a common beginners mistake.

How to Debug

Practice. Practice. Practice. You will see errors or unexpected behavior in your programs and then have to figure out why the thing that is happening is happening. There are a few common types of bugs you might run into.

Syntax Errors

These are usually the easiest to fix. There are a lot of code editors out there, and most of them will support syntax highlighting as well code analysis to let you know that when you have written some code that it knows just will not work. If you are not using one of these more advanced editors then whatever is running your code will immediately inform you of syntax errors and stop trying to run your program. The solution is usually as simple as correcting your code.

Run Time Errors

These errors can be a little harder to track down. You will definitely see an error output in the console for these types of errors. And while the output will definitely tell you what happened, it can be harder to track down why it happened.

Logic Errors

These are definitely the trickiest to figure out. These present themselves as unexpected behavior, so you won't see any error output to the console.
