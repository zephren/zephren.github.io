The Basics

The Basics

Any programing language is about having a bunch of small pieces of knowledge about how the language works (and how programs work in general). These little piece of knowledge are often referred to as "tools". The more tools you have, the more you can do. You will be able to use these tools in ever increasing combinations with each other to produce better, smarter, and more efficient programs. So let's jump right into building up your box of tools; the first one of which you have already used.

Seeing Something Happen

This first tool has a lot more going on than I will cover initially, so I'll just ask that you use it as I am giving it to you and, for now, simply accept that it does what we need it to. It will be covered later more in depth once you understand some additional concepts.

console.log()

This line written as is will technically do something, though you wont actually be able to see the result. To make it ever so slightly more interesting let's put something between the parentheses.

console.log(42)

If you run this example in your editor, you will see the number 42 appear in your output.

All we did here was tell JS to output (or log) a number, and thats exactly what it did. You can log literally anything (except [keywords]) in JS, and this will be very handy when you find that your program is not working as you expect. We will cover more about console.log in future sections, but for now feel free to play with logging things to the console. Here are a few examples that will make more sense shortly:

console.log(3.14)

console.log('This is a string')

console.log(true)

Code in Your Head

Something that will come with experience will be the critically important ability to run/execute the code you are looking at in your head. This is just another tool that you will develop over time. If you are now able to look at a line like:

console.log('This puts something in the console')

And can then imagine that text appearing in your output, then you're already on your way.

???

You may ask yourself (or try to ask me, to which I will have no reply as I cannot hear you) things like:

1) Can I do it this way? 2) Why do I have to do it this way? 3) I don't like how this works, can I do it differently?

And to answer those questions:

1) Maybe, but probably not. 2) Because that's how it is done. 3) Again, maybe, there are a lot of ways to do the same thing in JavaScript, as you will learn over time.

I will attempt to make it clear why things are done the way they are and try to give you some ways to help remember these things.

Variables

Let me start with another metaphor. Imagine you have an index card. There are two things you can do with this index card:

1) Give it a title/name. 2) Write some information (also called a value) on it.

When you are writing on this index card you are using a pen for the name and a pencil for the value. Once you've written the name, it cannot be changed. However since you wrote the value in pencil, there is nothing stopping you from changing it whenever you want. Use that mental model for now as we talk about variables.

Name

The name of a variable can be whatever you want with a few restrictions:

1) It should only contain letters, numbers, and underscores. 2) It cannot begin with a number (but may contain numbers). 3) It cannot contain any spaces.

There are a few exceptions to other characters that you can use for variable names (you can even use emoji... try it out to get it out of your system, but please, never ever do it for a real program), but at this point there is no reason you should need to use anything other than letters and numbers. I won't event be using underscores in my examples.

You will see me use what is called camel case. This just refers to using multiple words in the name of a variable, removing the spaces, and capitalizing the first letter of each word after the first word. Here are a few examples:

  • theNumber
  • theBestString
  • aTacoTruck
  • samsCafe

Value

Here is a variable with just a name:

var myFirstVariable

Think of it as having no value, or that you have an index card with just a name at the top, but you haven't written any value down on it. You can either give the variable a value immediately after giving it a name, or you can set it aside and give it a value later. Either way, you will need to use the equals sign to assign a value to a variable. Here are both of those options, respectively:

var myFirstVariable = 42

var myFirstVariable
myfirstVariable = 42

Now would be a good time to mention that most code is executed from right to left. Meaning the value on the right of the equals sign will be assigned to the variable on the left. It absolutely will not work the other way, by having the value on the left and the variable name on the right.

Types

Numbers

There are two types of numbers: integers and floats. There are limits to the rang of numbers you can represent, so keep in mind if you try to use a number that is too large, too small, or even just right, you might see some weird behavior from your program.

Integer

An integer is any whole number. You can represent whole numbers well over a trillion, but if you get too high or too low, and add or subtract from those numbers, you'll notice that your value doesn't change at all. I'll leave it up to you to figure out what the actual integer limits are.

Float

A float is any number with some decimal place. You will definitely encounter The weirdness with floats at some point while running your code. To demonstrate this, run the following line:

console.log(2.2 + 1.1)

You'll see that what gets output it almost what you wanted, but computers have their limits with how they can represent numbers. Just like the integer has an upper an lower limit it can represent, floats have fractions of numbers that they can't quite represent. But they do try to get as close as possible.

Character

A character is a single letter or symbol (for example anything that you can see on your keyboard can be a character)

String

A string is a series of characters. If that sentence doesn't quite make sense, try to imagine popcorn lined up with a single strand of thread connecting each piece of popcorn. On each piece of popcorn you can write a single character. Together all the pieces of popcorn make up a string. This could be a sentence, a person's name, or a color. A string can be anything you can create by typing on your keyboard.

Boolean

A boolean is is a true or false value.

Math

If you ran the console.log above, you've already done a bit of math in JS. Math in most programming languages is the same, with the same operators that you might have used in a math class: + - * / ( ). There is a concept that is worth understanding called [order of operations]. This is simply the order in which each mathematical operator gets used. Rather than trying to explain this concept to you, try playing around with adding, subtracting, multiplying, and dividing some numbers. If you don't get the result you might have expected try figuring out what order things happened in to give you the result you got, and then add some parentheses to get the result you were originally expecting. Heres an example:

console.log(1 + 2 * 3)

You might expect this to give you 9:

1) 1 + 2 = 3 2) 3 * 3 = 9

But you actually get 7. What is happening is that the multiplication is occurring before the addition:

1) 2 * 3 = 6 2) 1 + 6 = 7

To get 9 you simply need to add parentheses around the addition:

console.log((1 + 2) * 3)

Parentheses always take the highest precedence when doing math.
